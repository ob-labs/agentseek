# Blog

The blog is the narrative layer of the site.

Use it for content that is better explained as context, intent, or workflow rather than as a
reference table.

## Editorial tone

The reference pages answer "how does this work?" The blog should answer "why this shape?", "why
now?", and "how should I approach this capability in practice?"

## Suggested first posts

- an introduction to agentseek as a database-native agent runtime
- a map of the contrib packages and where each one fits
- a story from local chat to gateway and storage-backed workflows
