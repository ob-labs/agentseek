# Configuration

bubseek uses a single manifest file, `bubseek.toml`, as the source of truth for the distribution. The lockfile `bubseek.lock` is generated by `bubseek lock` and should be committed for reproducible syncs.

## Manifest: `bubseek.toml`

### `[project]`

Metadata for the distribution (name and version).

```toml
[project]
name = "bubseek"
version = "0.0.1"
```

### `[bub]`

Where to obtain the Bub runtime. You can use a **local path** or a **remote git source**.

**Remote (recommended):**

```toml
[bub]
repo = "https://github.com/bubbuild/bub"
ref = "frost/bub-framework"
```

**Local path** (e.g. for development):

```toml
[bub]
path = "../bub"
```

### `[[contrib]]`

Contrib packages (Bub plugins or add-ons) from one or more repositories. Each entry can specify `name`, `repo`, `path` (subdirectory inside the repo), and `ref`. Alternatively, use a single `source` URL (e.g. `git+https://...@branch`).

```toml
[[contrib]]
name = "bub-codex"
repo = "https://github.com/bubbuild/bub-contrib"
path = "packages/bub-codex"
ref = "main"

[[contrib]]
name = "bub-schedule"
repo = "https://github.com/bubbuild/bub-contrib"
path = "packages/bub-schedule"
ref = "main"
```

If only `name` is given, defaults are applied (e.g. `repo` and `path = "packages/<name>"` from the default contrib repo).

### `[[skills]]`

Skills are directories containing a `SKILL.md` and related files. They can be **local** (path relative to the config) or **remote** (repo + path + ref or a `source` URL).

**Local (bundled):**

```toml
[[skills]]
name = "bubseek-bootstrap"
path = "skills/bubseek-bootstrap"
```

**Remote:**

```toml
[[skills]]
name = "remote-example"
repo = "https://github.com/example/skills-repo"
path = "skills/remote-example"
ref = "v1.2.3"
```

During `bubseek sync`, local skills are copied from the distribution; remote skills are cloned (or fetched from cache) and then copied into the workspace.

## Lockfile: `bubseek.lock`

Generated by `bubseek lock`. It records:

- **Remote sources** — Resolved git commit SHA and source identifier for reproducibility.
- **Local entries** — Content hash (e.g. directory or package hash) so changes are detected.

`bubseek sync` verifies the lock against the current `bubseek.toml` (e.g. config checksum) before installing. Do not edit the lockfile by hand; regenerate it with `bubseek lock` after config changes.

## Source model summary

| Kind   | In config                    | In lock                         |
|--------|------------------------------|---------------------------------|
| Local  | `path`                       | `kind = "local"`, path, sha256  |
| Remote | `repo` + `ref` or `source`   | `kind = "remote"`, source, resolved_commit, sha256 |

Remote git sources are cached under a shared directory (see [Architecture](architecture.md#cache)).
